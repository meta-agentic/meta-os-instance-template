# Project Registry

One node per repo the OS covers. A node **points at** its repo (path, purpose, stack,
entry points, and — per [[systems/ontology.yaml|the ontology]] — its `repo:` remote) — it
never duplicates the code. This is the meta-OS's map of the estate.

## Adding a project

Copy [[templates/project|the project template]] to `projects/<name>.md`, fill it in, and
add a row above. Keep the node in sync with reality — it's the OS's ground truth for
"what is this repo and how do I enter it."
